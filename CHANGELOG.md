## [Unreleased]

## [0.3.2] - 2026-04-29

### 新增
- 新增 APT 仓库签名辅助脚本，可初始化专用 GPG 签名密钥并导出安装用公钥
- 新增 GitHub Pages APT 发布工作流，用于构建、签名并发布 Debian/Ubuntu 仓库
- 新增容器化 `.deb` 构建路径，方便在 macOS 或非 Debian 主机上生成 Linux 安装包

### 变更
- 新增 `aiproxyhub.py` 作为新的顶层推荐启动入口，`start.py` 与 `router_server.py` 继续保留为兼容别名
- Web 底部项目信息将收敛为更轻量的 footer 展示
- README、发布流程和外部测试文档的示例命令更新到 `0.3.2`

### 修复
- 上游 API 进入冷却、订阅临时冻结或暂时不可用时，Web 启用开关会按运行态显示为关闭
- 用户可在自动路由冷却期间手动重新启用上游，不必等待原冷却倒计时结束
- 测试连接成功或真实请求成功后，会自动清理对应上游的冷却和订阅临时冻结状态
- 修改故障冷却秒数后，会基于最后失败时间重新计算已有冷却窗口，不再沿用旧倒计时

## [0.3.0] - 2026-04-14

### 新增
- Web 与 CLI 统一展示项目元数据，包括版本、作者、许可证、更新方式与源码来源占位信息
- 远程 Linux 烟测脚本新增 SSH 私钥、SSH 选项与真实服务健康检查能力
- 发布文档新增外部 Linux / Windows 测试说明，其中 Windows 可使用虚拟机作为验证目标

### 变更
- README、发布流程和外部测试文档统一改为面向公开发布的表达方式
- 贡献指南去掉仓库所有者占位符，改为适用于正式仓库发布前后的通用说明
- `rich` 依赖上限收紧到 `<14`，降低与现有 `gemini-cli` 环境共装时的冲突概率

### 修复
- 移除日志中的占位发布日期，避免对外形成错误版本时间线
